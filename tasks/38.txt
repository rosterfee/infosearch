<p><b>AmigaOS</b>&#160;— <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">операционная система</a> для компьютеров семейства <a href="/wiki/Amiga" title="Amiga">Amiga</a>, имеет атипичное <a href="/wiki/%D0%9C%D0%B8%D0%BA%D1%80%D0%BE%D1%8F%D0%B4%D1%80%D0%BE" title="Микроядро">микроядро</a>. Классическую AmigaOS принято рассматривать как совокупность двух составляющих: Kickstart и Workbench. Kickstart обеспечивает абстрагирование от уникального аппаратного обеспечения Amiga и содержит в себе: планировщик вытесняющей многозадачности (Exec), дисковую операционную систему (AmigaDOS) и библиотеки графического интерфейса (Intuition). Workbench является графическим интерфейсом пользователя, и представлен как правило одноимённым рабочим столом или другим файловым менеджером.
</p>

<h2><span id=".D0.9A.D1.80.D0.B0.D1.82.D0.BA.D0.B8.D0.B9_.D0.BE.D0.B1.D0.B7.D0.BE.D1.80"></span><span class="mw-headline" id="Краткий_обзор">Краткий обзор</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS&amp;veaction=edit&amp;section=1" class="mw-editsection-visualeditor" title="Редактировать раздел «Краткий обзор»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AmigaOS&amp;action=edit&amp;section=1" title="Редактировать раздел «Краткий обзор»">править код</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>AmigaOS основана на технологии совместно используемых <a href="/wiki/%D0%91%D0%B8%D0%B1%D0%BB%D0%B8%D0%BE%D1%82%D0%B5%D0%BA%D0%B0_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Библиотека (программирование)">библиотек</a> (файлы с расширением <tt>.library</tt>), что обеспечивает её модульность и расширяемость. Библиотеки представляют собой сборники функций, в том числе и тех, которые обеспечивают функционирование самой операционной системы.
</p><p>В этом плане библиотеки AmigaOS аналогичны библиотечным технологиям других <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">операционных систем</a>, но обладают также свойствами совместного использования и переместимости (решение имеет свои достоинства и недостатки).
</p><p>Изначально предполагалось, что единственная библиотека, которая имеет жёстко заданный адрес в памяти и не может обновляться пользователем&#160;— exec.library по адресу <tt>4</tt>. Впоследствии развитие аппаратной части <a href="/wiki/Amiga" title="Amiga">Amiga</a> сторонними компаниями и банкротство корпорации <a href="/wiki/Commodore" title="Commodore">Commodore</a> привели к появлению решений (<a href="/w/index.php?title=%D0%90%D0%BA%D1%81%D0%B5%D0%BB%D0%B5%D1%80%D0%B0%D1%82%D0%BE%D1%80%D1%8B_Amiga&amp;action=edit&amp;redlink=1" class="new" title="Акселераторы Amiga (страница отсутствует)">акселераторы</a> и <a href="/w/index.php?title=%D0%AD%D0%BC%D1%83%D0%BB%D1%8F%D1%86%D0%B8%D1%8F_AmigaOS&amp;action=edit&amp;redlink=1" class="new" title="Эмуляция AmigaOS (страница отсутствует)">эмуляторы</a>), позволяющих перемещать и обновлять также и exec.library (так называемый «механизм затенения»). Эта библиотека является <a href="/wiki/%D0%AF%D0%B4%D1%80%D0%BE_%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%BE%D0%B9_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D1%8B" title="Ядро операционной системы">ядром</a> AmigaOS.
</p><p><a href="/wiki/%D0%94%D1%80%D0%B0%D0%B9%D0%B2%D0%B5%D1%80" title="Драйвер">Драйверы</a> AmigaOS (файлы с суффиксом имени файла <tt>.device</tt>), также являются библиотеками функций, обслуживающих специфичные для подключаемой периферии и устройств Amiga-вызовы, и имеющие стандартный интерфейс запросов. Существует также специфичное для AmigaOS понятие «<i>обработчика</i>» (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a>&#160;<span lang="en" style="font-style:italic;">handler</span>)&#160;— драйвера, имеющего более высокий уровень абстракции. Обработчики обеспечивают управление распределением ресурсов Amiga и представляют собой <a href="/wiki/%D0%9C%D0%BD%D0%BE%D0%B3%D0%BE%D0%B7%D0%B0%D0%B4%D0%B0%D1%87%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Многозадачность">процессы</a>, запускаемые при старте AmigaOS.
</p><p>Один из важнейших обработчиков AmigaOS&#160;— обработчик <a href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Файловая система">файловой системы</a>. AmigaOS может использовать любую файловую систему, для которой был написан обработчик. Эта возможность используется в <a href="/wiki/CrossDOS" title="CrossDOS">CrossDOS</a>&#160;— приложении, служащем для обмена файлами, например с <a href="/wiki/File_Allocation_Table" class="mw-redirect" title="File Allocation Table">FAT</a>,&#160;— и его аналогах. Для AmigaOS существуют обработчики для всех наиболее распространённых файловых систем, а также некоторых экзотических. У пользователей наиболее популярны следующие обработчики файловых систем: <a href="/w/index.php?title=OFS&amp;action=edit&amp;redlink=1" class="new" title="OFS (страница отсутствует)">OFS</a>, <a href="/w/index.php?title=Fast_file_system&amp;action=edit&amp;redlink=1" class="new" title="Fast file system (страница отсутствует)">FFS</a>, <a href="/w/index.php?title=FFS2&amp;action=edit&amp;redlink=1" class="new" title="FFS2 (страница отсутствует)">FFS2</a>, <a href="/w/index.php?title=PFS&amp;action=edit&amp;redlink=1" class="new" title="PFS (страница отсутствует)">PFS</a>, <a href="/w/index.php?title=PFS2&amp;action=edit&amp;redlink=1" class="new" title="PFS2 (страница отсутствует)">PFS2</a>, <a href="/wiki/Smart_file_system" class="mw-redirect" title="Smart file system">SFS</a>.
</p><p>Как правило, обработчики передают в <a href="/wiki/DOS" title="DOS">DOS</a> название устройства, связанного с ними, но существуют и исключения. Например, обработчик Speak-handler (соответствующий устройству SPEAK:) позволяет управлять программным синтезатором речи (аналогичным реализации в <a href="/wiki/OS/2_Warp" class="mw-redirect" title="OS/2 Warp">OS/2 Warp</a>, и более поздней реализации в <a href="/wiki/Windows_NT" title="Windows NT">Windows NT</a>) и является одним из примеров службы, не связанной с периферией.
</p><p>AmigaOS безразличен регистр, в котором набрано название устройства (принято использовать прописные символы), а идентификатором устройства служит символ двоеточия (:) после его имени. После двоеточия может следовать дополнительная информация для обработчика, указывающая на объект и способ управления. Для обработчика файловой системы, это, как правило, информация о пути к файлу. Для других обработчиков это может быть указание потока ввода-вывода (например, устройство SER:, связанное с последовательным портом, воспринимает данные о битрейте, стартовом/стоповом бите и&#160;т.&#160;д.).
</p><p>Файловые системы используют в названиях устройств их порядковые номера в системе. Например, устройство <tt>DF0:</tt> по умолчанию связано с первым дисководом в системе, а <tt>DH0:</tt>&#160;— с первым разделом винчестера.
</p><p>Помимо этого, обработчики устройств могут неограниченно связываться друг с другом. В этом случае путь к устройству может быть достаточно «извилистым». Метки томов устройств также считаются устройствами. Например, если дискета в дисководе <tt>DF0:</tt> имеет метку тома Workbench, то пользователь может обратиться к ней как к логическому устройству <tt>Workbench:</tt>.
</p><p>Если пользователь хочет обратиться к файлу Bar, находящемуся в каталоге Foo на диске Work в дисководе <tt>DF0:</tt>, он может записать это так:
</p>
<pre>DF0:Foo/Bar
</pre>
<p>или так:
</p>
<pre>Work:Foo/Bar
</pre>
<p>Однако эти формы записи не полностью аналогичны. Последняя форма используется, когда системе уже известно, что дискета с именем «Work»&#160;— это не любая дискета в дисководе <tt>DF0:</tt>. Поэтому, если попытаться вставить в тот же дисковод <tt>DF0:</tt> копию нашей дискеты, но с другой меткой тома и обратиться к ней как Work:Foo/Bar, система обнаружит ошибку и выдаст сообщение:
</p>
<pre>Please insert volume "Work" in any drive
</pre>
<p>Приложениям AmigaOS часто требуется получить доступ к файлу, не имея данных о его месторасположении (включая название устройства и метку тома): им известен только его тип, является ли файл библиотекой, документацией, локализацией сообщений программы и&#160;т.&#160;д. Этот вопрос решается на уровне стандартных логических устройств, связанных с файлами определённых типов. Поиск соответствующего файла производится в первую очередь в этих устройствах. По умолчанию в AmigaOS приняты следующие логические устройства:
</p>
<ul><li><tt>SYS:</tt>, указывающее на носитель, с которого была загружена система. Создаётся автоматически при старте AmigaOS</li>
<li><tt>LIBS:</tt>, указывающее на каталог, содержащий системные библиотеки. Обычно, <tt>SYS:Libs/</tt></li>
<li><tt>DEVS:</tt>, указывающее на каталог, содержащий драйвера. Обычно, <tt>SYS:Devs/</tt></li>
<li><tt>L</tt>, указывающее на каталог, содержащий хэндлеры. Обычно, <tt>SYS:L/</tt></li>
<li><tt>C:</tt>, указывающее на каталог, содержащий команды Shell. Обычно, <tt>SYS:C/</tt></li></ul>
<h2><span id=".D0.A1.D0.BE.D1.81.D1.82.D0.B0.D0.B2.D0.BB.D1.8F.D1.8E.D1.89.D0.B8.D0.B5_AmigaOS"></span><span class="mw-headline" id="Составляющие_AmigaOS">Составляющие AmigaOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS&amp;veaction=edit&amp;section=2" class="mw-editsection-visualeditor" title="Редактировать раздел «Составляющие AmigaOS»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AmigaOS&amp;action=edit&amp;section=2" title="Редактировать раздел «Составляющие AmigaOS»">править код</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Классическую AmigaOS принято рассматривать как совокупность двух составляющих: Kickstart и Workbench. До AmigaOS 3.5 они считались неотделимыми и зависимыми друг от друга.
</p><p>Kickstart (в версиях от V0 до V42) предполагал использование определённых версий Workbench. Например, на <a href="/wiki/Amiga_500" title="Amiga 500">Amiga 500</a>, имеющей Kickstart V36, может исполняться Workbench 1.3, а корректная загрузка, например, Workbench 3.1 невозможна. Исключением из этого правила является только Workbench 2.1, который может быть загружен на Amiga, имеющей более ранний Kickstart V37. Последние версии Workbench&#160;— 3.5 и 3.9&#160;— используют Kickstart V40 для первоначальной загрузки, после чего происходит его обновление до V42 (или V44) в специально выделенной и защищённой области памяти (функция замещения ПЗУ у акселераторов).
</p>
<h3><span class="mw-headline" id="Kickstart">Kickstart</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS&amp;veaction=edit&amp;section=3" class="mw-editsection-visualeditor" title="Редактировать раздел «Kickstart»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AmigaOS&amp;action=edit&amp;section=3" title="Редактировать раздел «Kickstart»">править код</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote navigation-not-searchable noprint dabhide">У этого термина существуют и другие значения, см. <a href="/wiki/Kickstart_(%D0%B7%D0%BD%D0%B0%D1%87%D0%B5%D0%BD%D0%B8%D1%8F)" class="mw-disambig" title="Kickstart (значения)">Kickstart (значения)</a>.</div>
<p><b>Kickstart</b>&#160;— название, принятое для <a href="/wiki/%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%BD%D0%BE%D0%B5_%D0%9F%D0%9E" class="mw-redirect" title="Системное ПО">системного ПО</a>, осуществляющего <a href="/wiki/%D0%97%D0%B0%D0%B3%D1%80%D1%83%D0%B7%D1%87%D0%B8%D0%BA_%D0%9E%D0%A1" class="mw-redirect" title="Загрузчик ОС">первоначальную загрузку</a> <a href="/wiki/Amiga" title="Amiga">Amiga</a>. На <a href="/wiki/Amiga_1000" title="Amiga 1000">Amiga 1000</a> (первой выпущенной модели), Kickstart загружался с 3,5″-диска и занимал 256 Кб. В более поздних моделях Kickstart был перенесён в <a href="/wiki/%D0%9F%D0%BE%D1%81%D1%82%D0%BE%D1%8F%D0%BD%D0%BD%D0%BE%D0%B5_%D0%B7%D0%B0%D0%BF%D0%BE%D0%BC%D0%B8%D0%BD%D0%B0%D1%8E%D1%89%D0%B5%D0%B5_%D1%83%D1%81%D1%82%D1%80%D0%BE%D0%B9%D1%81%D1%82%D0%B2%D0%BE" title="Постоянное запоминающее устройство">постоянное запоминающее устройство</a>, а его объём вырос до 512 Кб. Все версии Kickstart <a href="/wiki/%D0%9E%D0%B1%D1%80%D0%B0%D1%82%D0%BD%D0%B0%D1%8F_%D1%81%D0%BE%D0%B2%D0%BC%D0%B5%D1%81%D1%82%D0%B8%D0%BC%D0%BE%D1%81%D1%82%D1%8C" title="Обратная совместимость">обратно совместимы</a>, несмотря на многочисленные изменения, происходившие в <a href="/wiki/%D0%A7%D0%B8%D0%BF%D1%81%D0%B5%D1%82" title="Чипсет">чипсете</a> классической Amiga с появлением каждой новой модели.
</p><p>Помимо кода, отвечающего за первоначальную загрузку Workbench, Kickstart также содержит значительную часть самой AmigaOS. Эта часть носит название <b>Amiga ROM Kernel</b> и включает в себя Intuition (библиотеки, обеспечивающие <a href="/wiki/%D0%93%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81" class="mw-redirect" title="Графический интерфейс">GUI</a>), Exec (микроядро, реализующее вытесняющую <a href="/wiki/%D0%9C%D0%BD%D0%BE%D0%B3%D0%BE%D0%B7%D0%B0%D0%B4%D0%B0%D1%87%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Многозадачность">многозадачность</a>) и AmigaDOS (библиотеки, обеспечивающие работу с дисками). Последняя составляющая основана на <a href="/wiki/TRIPOS" title="TRIPOS">TRIPOS</a>, портированной специально для Amiga компанией <a href="/wiki/MetaComCo" title="MetaComCo">MetaComCo</a>, и написана на языке <a href="/wiki/BCPL" title="BCPL">BCPL</a> (считается «прародителем» языка <a href="/wiki/%D0%A1%D0%B8_(%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)" title="Си (язык программирования)">Си</a>).
</p><p>Последние версии Kickstart содержат драйвера для <a href="/wiki/ATA" title="ATA">IDE</a>- и <a href="/wiki/SCSI" title="SCSI">SCSI</a>-контроллеров, порта <a href="/wiki/PCMCIA" title="PCMCIA">PCMCIA</a> и различной периферии самой Amiga.
</p><p>С помощью <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Программное обеспечение">ПО</a> от сторонних производителей возможно замещение Kickstart, находящегося в <a href="/wiki/%D0%9F%D0%BE%D1%81%D1%82%D0%BE%D1%8F%D0%BD%D0%BD%D0%BE%D0%B5_%D0%B7%D0%B0%D0%BF%D0%BE%D0%BC%D0%B8%D0%BD%D0%B0%D1%8E%D1%89%D0%B5%D0%B5_%D1%83%D1%81%D1%82%D1%80%D0%BE%D0%B9%D1%81%D1%82%D0%B2%D0%BE" title="Постоянное запоминающее устройство">ПЗУ</a> Amiga, на Kickstart другой версии, находящийся в специально выделенной и защищённой области ОЗУ. Например, Kickstart V34 может использоваться владельцем компьютера, имеющего Kickstart V37, для более корректной работы старых игр. Программы, осуществляющие такое замещение, называются «софткикерами» (Softkickers). Также существуют аппаратные переключатели Kickstart, позволяющие использовать компьютер, как бы обладающий двумя чипсетами&#160;— устаревшим и штатным. Таким образом, существует множество средств для динамического замещения ПЗУ.
</p>
<h3><span class="mw-headline" id="Workbench">Workbench</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS&amp;veaction=edit&amp;section=4" class="mw-editsection-visualeditor" title="Редактировать раздел «Workbench»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AmigaOS&amp;action=edit&amp;section=4" title="Редактировать раздел «Workbench»">править код</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Workbench&#160;— штатная графическая оболочка AmigaOS. Название оболочки является метафорой словосочетания «рабочий стол» (дословно&#160;— «верстак»), поэтому аналогия была продолжена и дальше: каталоги изображены ящиками рабочего стола, исполняемые файлы&#160;— инструментами, данные&#160;— проектами, а остальные составляющие <a href="/wiki/%D0%93%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81" class="mw-redirect" title="Графический интерфейс">GUI</a>&#160;— разного рода приспособлениями. Большинство приложений AmigaOS использует всплывающие меню, традиционно начинающиеся со слова Project («Проект»), а не File («Файл»), как на других платформах. Строго говоря, термин Workbench относится только к основному файловому менеджеру AmigaOS, хотя часто используется для обозначения всей той её части, которая расположена вне <a href="/wiki/%D0%9F%D0%BE%D1%81%D1%82%D0%BE%D1%8F%D0%BD%D0%BD%D0%BE%D0%B5_%D0%B7%D0%B0%D0%BF%D0%BE%D0%BC%D0%B8%D0%BD%D0%B0%D1%8E%D1%89%D0%B5%D0%B5_%D1%83%D1%81%D1%82%D1%80%D0%BE%D0%B9%D1%81%D1%82%D0%B2%D0%BE" title="Постоянное запоминающее устройство">ПЗУ</a> (то есть вне Kickstart).
</p><p>Интерфейс Workbench похож на Finder в <a href="/wiki/Mac_OS" class="mw-redirect" title="Mac OS">Mac OS</a>: здесь также используется главное меню в виде полосы в верхней части экрана. В отличие от стандартной мыши <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a>, имеющей единственную кнопку, стандартная мышь классической <a href="/wiki/Amiga" title="Amiga">Amiga</a> имеет две кнопки. Удержанием правой кнопки открывается всплывающее меню, отпусканием&#160;— выбирается опция. Левая кнопка мыши используется для управления открытыми на рабочем столе окнами (перемещение, свёртывание, закрытие, выделение и&#160;т.&#160;п.). Позднее была добавлена поддержка мыши с бо́льшим числом кнопок и колесом.
</p>
<h3><span class="mw-headline" id="Exec">Exec</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS&amp;veaction=edit&amp;section=5" class="mw-editsection-visualeditor" title="Редактировать раздел «Exec»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AmigaOS&amp;action=edit&amp;section=5" title="Редактировать раздел «Exec»">править код</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><span data-interwiki-lang="en" data-interwiki-article="Exec (Amiga)"><a href="/w/index.php?title=Exec&amp;action=edit&amp;redlink=1" class="new" title="Exec (страница отсутствует)">Exec</a></span><span class="noprint" style="font-style:normal; font-weight:normal;"><sup><a href="https://en.wikipedia.org/wiki/Exec_(Amiga)" class="extiw" title="en:Exec (Amiga)"><span title="Exec (Amiga) — версия статьи «Exec» на английском языке">[en]</span></a></sup></span> (ExegSG начиная от AmigaOS 4) является многозадачным <a href="/wiki/%D0%9C%D0%B8%D0%BA%D1%80%D0%BE%D1%8F%D0%B4%D1%80%D0%BE" title="Микроядро">микроядром</a> AmigaOS. Exec обеспечивает функциональность для <a href="/wiki/%D0%9C%D0%BD%D0%BE%D0%B3%D0%BE%D0%B7%D0%B0%D0%B4%D0%B0%D1%87%D0%BD%D0%BE%D1%81%D1%82%D1%8C" title="Многозадачность">многозадачности</a>, распределения памяти, обработки прерываний и обработки <a href="/wiki/%D0%91%D0%B8%D0%B1%D0%BB%D0%B8%D0%BE%D1%82%D0%B5%D0%BA%D0%B0_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Библиотека (программирование)">динамических разделяемых библиотек</a>. Оно выступает в качестве планировщика для задач, запущенных в системе, обеспечивающего преимущественную многозадачность с приоритетами планирования в <a href="/wiki/%D0%A1%D1%80%D0%B5%D0%B4%D0%B0_%D1%80%D0%B0%D0%B1%D0%BE%D1%87%D0%B5%D0%B3%D0%BE_%D1%81%D1%82%D0%BE%D0%BB%D0%B0" title="Среда рабочего стола">графическом окружении</a>. Exec также обеспечивает доступ к другим библиотекам и на высоком уровне обеспечивает связи между процессами через передачу сообщений. Другие сопоставимые микроядра имели проблемы производительности из-за необходимости копирования сообщений между <a href="/wiki/%D0%90%D0%B4%D1%80%D0%B5%D1%81%D0%B0%D1%86%D0%B8%D1%8F_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D0%B8" title="Адресация памяти">адресными пространствами</a>. Так как <a href="/wiki/Amiga" title="Amiga">Amiga</a> имеет только одно <a href="/wiki/%D0%90%D0%B4%D1%80%D0%B5%D1%81%D0%B0%D1%86%D0%B8%D1%8F_%D0%BF%D0%B0%D0%BC%D1%8F%D1%82%D0%B8" title="Адресация памяти">адресное пространство</a>, наличие Exec является довольно эффективным инструментом для передачи сообщений в системе.
</p>
<h3><span class="mw-headline" id="AmigaDOS">AmigaDOS</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS&amp;veaction=edit&amp;section=6" class="mw-editsection-visualeditor" title="Редактировать раздел «AmigaDOS»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AmigaOS&amp;action=edit&amp;section=6" title="Редактировать раздел «AmigaDOS»">править код</a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a class="mw-selflink-fragment" href="#AmigaDOS">AmigaDOS</a> обеспечивает системную часть в <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">операционной системе</a> AmigaOS. Это включает в себя <a href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Файловая система">файловую систему</a>, управление файлами и каталогами, <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81_%D0%BA%D0%BE%D0%BC%D0%B0%D0%BD%D0%B4%D0%BD%D0%BE%D0%B9_%D1%81%D1%82%D1%80%D0%BE%D0%BA%D0%B8" title="Интерфейс командной строки">интерфейс командной строки</a>, перенаправление файлов, консольные окна, и так далее. <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81" title="Интерфейс">Интерфейс</a> системы многофункционален, он включает в себя перенаправление команд, эффективную передачу сообщений, сценарии с шаблоном примитивов программирования, и систему глобальных и локальных <a href="/wiki/%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D0%B0%D1%8F_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Переменная (программирование)">переменных</a>.
</p><p>В AmigaOS первого поколения AmigaDOS был основан на <a href="/wiki/TRIPOS" title="TRIPOS">TRIPOS</a>, который был написан на языке программирования <a href="/wiki/BCPL" title="BCPL">BCPL</a>. Взаимодействие с ним других языков оказалось задачей трудной и приводило порой к ошибкам, в связи с чем порт <a href="/wiki/TRIPOS" title="TRIPOS">TRIPOS</a> был признан не очень эффективным.
</p><p>В AmigaOS второго поколения AmigaDOS был переписан на <a href="/wiki/%D0%AF%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F" title="Язык программирования">язык программирования</a> <a href="/wiki/%D0%A1%D0%B8_(%D1%8F%D0%B7%D1%8B%D0%BA_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F)" title="Си (язык программирования)">C</a> и <a href="/wiki/%D0%90%D1%81%D1%81%D0%B5%D0%BC%D0%B1%D0%BB%D0%B5%D1%80" title="Ассемблер">ассемблер</a>, сохранив совместимость с <a href="/wiki/BCPL" title="BCPL">BCPL</a>. В него были включены полная программа первого поколения и часть программы из третьей партии <a href="/w/index.php?title=AmigaDOS_Resource_Project&amp;action=edit&amp;redlink=1" class="new" title="AmigaDOS Resource Project (страница отсутствует)">AmigaDOS Resource Project</a> (ARP), которые уже нашли замену для многих из программных <a href="/wiki/BCPL" title="BCPL">BCPL</a>-утилит и интерфейсов.
</p><p>В последующем ARP также предоставил один из первых стандартизированных реквестеров файлов для <a href="/wiki/Amiga" title="Amiga">Amiga</a> с целью введения в рабочую среду <a href="/wiki/Amiga" title="Amiga">Amiga</a> для использования более дружественных <a href="/wiki/UNIX" class="mw-redirect" title="UNIX">UNIX</a>-стилю шаблонов (форматов) функций в параметрах командной строки. Другими нововведениями были существенные улучшения в диапазоне форматов файлов, реализуемых системой как объекты <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">ОС</a>, что позволяло загружать в память ту или иную информацию только один раз, сохранять её в памяти и значительно уменьшить время загрузки системы для последующих применений.
</p><p>В AmigaOS 4.0 в составе <a href="/wiki/DOS" title="DOS">DOS</a> отказались от наследия <a href="/wiki/BCPL" title="BCPL">BCPL</a> полностью, и, начиная от AmigaOS 4.1, она была переписана целиком с полной поддержкой <a href="/wiki/64_%D0%B1%D0%B8%D1%82%D0%B0" title="64 бита">64-битной архитектуры</a>.
</p><p>Само собой разумеется, что различные расширения часто используются в AmigaOS, но они не являются обязательными, и они не обрабатываются особым образом через <a href="/wiki/DOS" title="DOS">DOS</a>, которая ныне в системе относится лишь к каталогам и файлам. Исполняемые программы реализуются с использованием так называемого <a href="/wiki/%D0%9C%D0%B0%D0%B3%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%BE%D0%B5_%D1%87%D0%B8%D1%81%D0%BB%D0%BE_(%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5)" title="Магическое число (программирование)">магического числа или сигнатуры данных</a>.
</p>
<h3><span class="mw-headline" id="ARexx">ARexx</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=AmigaOS&amp;veaction=edit&amp;section=7" class="mw-editsection-visualeditor" title="Редактировать раздел «ARexx»">править</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=AmigaOS&amp;action=edit&amp;section=7" title="Редактировать раздел «ARexx»">править код</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>AmigaOS содержит поддержку интерпретируемого языка программирования <a href="/wiki/REXX" title="REXX">REXX</a> (<i><a href="/wiki/ARexx" title="ARexx">ARexx</a></i>&#160;— официальное сокращение словосочетания <i>Amiga Rexx</i>). Эта поддержка буквально «пронизывает» все части операционной системы и позволяет: писать сценарии (script) управления <a href="/wiki/%D0%9E%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0" title="Операционная система">ОС</a> (аналогично <a href="/wiki/Bash" title="Bash">bash</a> в <a href="/wiki/GNU/Linux" title="GNU/Linux">GNU/Linux</a>), управлять приложениями (поздний аналог&#160;— интеграция <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">VBA</a> в <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>), а также осуществлять обмен сообщениями между приложениями, поддерживающими <a href="/wiki/ARexx" title="ARexx">ARexx</a>-порты (аналог&#160;— <a href="/wiki/D-Bus" title="D-Bus">D-Bus</a>).
</p><p>Приложения могут открывать <a href="/wiki/ARexx" title="ARexx">ARexx</a>-порты и ожидать сообщения в виде текста от других приложений или ОС. Эти сообщения могут интерпретироваться таким образом, как если бы пользователь использовал <a href="/wiki/%D0%93%D1%80%D0%B0%D1%84%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81_%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D1%8F" title="Графический интерфейс пользователя">GUI</a> или <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81_%D0%BA%D0%BE%D0%BC%D0%B0%D0%BD%D0%B4%D0%BD%D0%BE%D0%B9_%D1%81%D1%82%D1%80%D0%BE%D0%BA%D0%B8" title="Интерфейс командной строки">командную строку</a> для непосредственного управления приложением. Например, <a href="/wiki/ARexx" title="ARexx">ARexx</a>-программа (обычный текстовый файл) может запустить почтовый клиент, сохранить полученные письма, вызвать программу для их обработки, а затем вызывать внешний просмотрщик текста для их отображения. Таким образом, <a href="/wiki/ARexx" title="ARexx">ARexx</a> позволяет одним приложениям управлять другими, перебрасывая данные через память, вместо создания множества маленьких файлов и перенесения дисковых операций по управлению ими на плечи пользователя.
</p>

В декабре <a href="/wiki/2007_%D0%B3%D0%BE%D0%B4" title="2007 год">2007 года</a> была портирована на классические A1200, A4000, A4000T, оснащённые <a href="/wiki/PowerPC" title="PowerPC">PPC</a>-акселераторами. Совместима с ранними версиями AmigaOS и <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%BD%D0%BE%D0%B5_%D0%BE%D0%B1%D0%B5%D1%81%D0%BF%D0%B5%D1%87%D0%B5%D0%BD%D0%B8%D0%B5" title="Программное обеспечение">ПО</a>, написанным для них с помощью встроенного JIT-эмулятора 68k-кода.</li></ul>
<ul><li>AmigaOS 4.1&#160;— Kickstart V51.28, V51.1&#160;— A1200, A4000, A4000T, оснащённые PPC-акселераторами, AmigaOne, Sam 440. Вышла в сентябре 2008 года в результате тесного взаимодействия и сотрудничества компаний <a href="/wiki/Hyperion_Entertainment" title="Hyperion Entertainment">Hyperion Entertainment</a> и <a href="/w/index.php?title=ACube_Systems&amp;action=edit&amp;redlink=1" class="new" title="ACube Systems (страница отсутствует)">ACube Systems</a> SRL. В 2009 году портирована на <a href="/wiki/Pegasos_II" class="mw-redirect" title="Pegasos II">Pegasos II</a>. Также существуют планы по адаптации для <a href="/wiki/Mac_mini" title="Mac mini">Mac mini</a> G4 (проект Moana). Регулярное обновление системы проходит автоматически через Интернет (в 2019&#160;г. уровень V53-54). Последнее публичное обновление AmigaOS4.1FE-Update2 - v53.14 (12.01.2021). Последний Kernel v54.56 (26.01.2023).</li></ul>
<ul><li>AmigaAnywhere&#160;— ранее называлась AmigaDE. Кроссплатформенная система, ориентированная на <a href="/wiki/%D0%9A%D0%B0%D1%80%D0%BC%D0%B0%D0%BD%D0%BD%D1%8B%D0%B9_%D0%BF%D0%B5%D1%80%D1%81%D0%BE%D0%BD%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D0%BA%D0%BE%D0%BC%D0%BF%D1%8C%D1%8E%D1%82%D0%B5%D1%80" title="Карманный персональный компьютер">КПК</a>, <a href="/wiki/%D0%A1%D0%BC%D0%B0%D1%80%D1%82%D1%84%D0%BE%D0%BD" title="Смартфон">смартфоны</a> и другие устройства домашнего использования. Отличительная черта&#160;— наличие переносимого <a href="/wiki/%D0%AF%D0%B4%D1%80%D0%BE_%D0%BE%D0%BF%D0%B5%D1%80%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%BD%D0%BE%D0%B9_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D1%8B" title="Ядро операционной системы">ядра</a>, похожего по своим возможностям на <a href="/wiki/JVM" class="mw-redirect" title="JVM">виртуальные Java-машины</a>. Программа, скомпилированная для AmigaAnywhere один раз, будет работоспособна на всех системах с установленной AmigaAnywhere (ряд КПК продаются с предустановленной AA) или AmigaAnywhere Player (существует для <a href="/wiki/GNU/Linux" title="GNU/Linux">GNU/Linux</a>, <a href="/wiki/Microsoft_Windows" class="mw-redirect" title="Microsoft Windows">Microsoft Windows</a> и <a href="/wiki/Mac_OS" class="mw-redirect" title="Mac OS">Mac OS</a>). Компания Hyperion Entertainment<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> (разработчик AmigaOS, сегодня) имеет планы в будущем включать AmigaAnywhere в AmigaOS 4.x.</li></ul>
